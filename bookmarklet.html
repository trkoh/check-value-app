ヴァルール確認ブックマークレット

下のボタンを、ブラウザのブックマークバーに ドラッグ してください。
以降は任意のWebページで、このブックマークをクリックすると、そのページで一番大きい画像のヴァルール変換結果が新規タブで開きます。

▶ ヴァルール確認

クリックすると現ページに画像が無い旨のアラートが出るのは正常です（このページに <img> が無いため）。

v2 への更新点 (2026-05-18)

Pinterest など CORS 非対応の画像ホスト（i.pinimg.com 等）で旧版が動かなかったため、ロジックを変更:

  1. ページ内の最大の <img> を選択して execCommand('copy') で 画像データそのものをクリップボードへコピー（右クリック→「画像をコピー」と同じ機構なので、CORS を経由しない）
  2. 本ツールを ?paste=1 で開く
  3. 本ツール側にペーストオーバーレイが出るので、ボタンを1回押すか Cmd/Ctrl+V でペースト完了

コピーが失敗した場合のみ、旧版同等の ?img=URL パスにフォールバックする。

古いブックマークを使っている場合は、再度ドラッグして上書きしてください。

ドラッグできない場合の手動設定

  1. 任意のページをブックマークする（後でURLを書き換えるため、何でも良い）
  2. ブックマークを編集 → 名前を「ヴァルール確認」に、URL を以下に置き換える
javascript:(function(){var i=Array.from(document.images).filter(function(x){return x.naturalWidth>50});if(!i.length){alert('画像が見つかりません');return}i.sort(function(a,b){return(b.naturalWidth*b.naturalHeight)-(a.naturalWidth*a.naturalHeight)});var im=i[0],u=im.currentSrc||im.src,c=false;try{var r=document.createRange();r.selectNode(im);var s=window.getSelection();s.removeAllRanges();s.addRange(r);c=document.execCommand('copy');s.removeAllRanges()}catch(e){}window.open('https://odayakalife.dev/check-value-app/'+(c?'?paste=1':'?img='+encodeURIComponent(u)),'_blank')})()

動作する条件 / しない条件

  • ✅ ページに <img> として埋め込まれた画像があれば拾える
  • ✅ Pinterest など CORS 非対応ホストでも動作（v2 でクリップボード経由に変更）
  • ❌ CSS の background-image で配置されている画像（Pinterest のグリッド画面など）は拾えない → 個別の画像詳細ページに飛んでから起動
  • ❌ サイトの Content Security Policy (CSP) で javascript: URL がブロックされていると無効（X (Twitter) など）→ 画像を新規タブで開いてから起動するか、手動で右クリック→画像をコピー→PWAで Cmd+V
  • ⚠️ execCommand('copy') は deprecated（廃止予定）扱いだが、Chrome/Safari/Firefox の現行版で動作する。将来 ClipboardItem API への移行が必要になる可能性

このページの仕組み

ブックマークレットは javascript: スキームの URL。ブラウザは URL バーやブックマークから javascript: が呼ばれた時、現在ページの window context で JS を実行する（MDN: javascript: URLs）。execCommand('copy') で選択中の <img> をコピーすると、ブラウザは内部的に画像のピクセルデータをクリップボードへ転送する（右クリック→画像をコピー と同じ機構）。これにより、画像ホストの CORS 設定に依存せずピクセルを移送できる。
