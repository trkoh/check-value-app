ヴァルール確認ブックマークレット

下のボタンを、ブラウザのブックマークバーに ドラッグ してください。
以降は任意のWebページで、このブックマークをクリックすると、そのページで一番大きい画像のヴァルール変換結果が新規タブで開きます。

▶ ヴァルール確認

クリックすると現ページに画像が無い旨のアラートが出るのは正常です（このページに `<img>` が無いため）。

ドラッグできない場合の手動設定

  1. 任意のページをブックマークする（後でURLを書き換えるため、何でも良い）
  2. ブックマークを編集 → 名前を「ヴァルール確認」に、URL を以下に置き換える
javascript:(function(){var i=Array.from(document.images);if(!i.length){alert('画像が見つかりません');return}i.sort(function(a,b){return(b.naturalWidth*b.naturalHeight)-(a.naturalWidth*a.naturalHeight)});var u=i[0].currentSrc||i[0].src;window.open('https://odayakalife.dev/check-value-app/?img='+encodeURIComponent(u),'_blank')})()

動作する条件 / しない条件

  • ✅ ページに <img> として埋め込まれた画像があれば拾える
  • ❌ CSS の background-image で配置されている画像（Pinterest のグリッド画面など）は拾えない → 個別の画像詳細ページに飛んでから起動
  • ❌ サイトの Content Security Policy (CSP) で javascript: URL がブロックされていると無効（X (Twitter) など）→ 画像を新規タブで開いてから起動するか、画像URLを手で ?img= に渡す
  • ⚠️ 渡された画像URLが CORS を許可していない場合、本ツール側で読込みが失敗する。サムネイル経由で済む場合は問題ないが、認証付き画像（要ログイン）は不可

このページの仕組み

ブックマークレットは javascript: スキームの URL。ブラウザは URL バーやブックマークから javascript: が呼ばれた時、現在ページの window context で JS を実行する。これは MDN: javascript: URLs に仕様化された挙動。ブックマークバーへドラッグ可能なのは、HTML の <a href="javascript:..."> の標準的なドラッグ挙動による（HTML Living Standard, dragstart）。
