ヴァルール変換

カラー画像を、明暗（ヴァルール）確認用のグレースケールに変換する。

入力

画像を画面にドラッグ＆ドロップ、または Cmd/Ctrl+V でペーストも可

出力モード

計算方式
Photoshop互換: 「モノクロ塗りつぶしレイヤー + Color ブレンドモード」を乗せたのと同じ計算。普段のヴァルール確認はこっち。 (BT.709 Y')

変換の詳細・根拠

ペイントアプリで「白カラーレイヤーを最上位に重ねる」と出るのと概ね同じ計算で、色を抜いて明暗の組み立てだけを残す。

BT.709 Y' (デフォルト)

Y' = 0.2126·R + 0.7152·G + 0.0722·B

係数は ITU-R BT.709 (HDTV/sRGBの輝度標準) で定められた赤・緑・青の重み。緑が最も明るく、青が最も暗く感じるという人間の目の特性に基づく。sRGBガンマ域で直接計算する Y' は、Photoshop / Procreate の "Color" ブレンドモードと概ね同等。

L* (CIE Lab)

sRGB → 線形 → 輝度 Y → L* = 116·f(Y) − 16 (D65)。知覚均等な明度尺度。Posterize / Notan の段階化と相性が良く、隣り合う段の知覚差が一定になる。

Posterize / Notan / Squint

選択した値空間で normalize（BT.709 は 0–255、L* は 0–100）し、N段に量子化。Notan は 2値（しきい 1/2）または 3値（しきい 1/3, 2/3）の特例。Squint は描画前に Gaussian blur を適用して「目を細めた構図」を再現する。

色を逆引き (Oklab)

出力グレーの明度を Oklab L にマップし、その L を固定して色相を 0–360° にスイープ。各色相で sRGB ガンメに収まる最大彩度を二分探索で求めて並べる。同じ「見かけの明度」で使える色候補が分かる。
